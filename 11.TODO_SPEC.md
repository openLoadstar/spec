# 📋 TODO_SPEC — TODO 시스템 규격 (v3.0)

## 설계 원칙

- **WayPoint가 Single Source of Truth**: TODO 목록은 WayPoint STATUS로부터 자동 산출한다. 별도 수동 등록 불필요.
- **sync 기반 동기화**: `loadstar todo sync`가 WayPoint 파일을 스캔하여 TODO_LIST를 갱신한다.
- **히스토리는 WayPoint에 보존**: 완료 이력은 WayPoint TECH_SPEC의 `[x] YYYY-MM-DD 항목명`을 직접 읽는다.
- **느슨한 관리**: BLOCKED 표시는 참고용이며 작업을 강제 차단하지 않는다.

## 데이터 구조

```
.clionly/TODO/
├── TODO_LIST.md          # sync가 관리하는 현재 작업 목록
└── WP_SNAPSHOT.json      # WP 파일 상태 캐시 (modTime, size, status)
```

> ⚠️ `.clionly/TODO/` 파일은 `loadstar todo` 명령으로만 조작한다. 직접 편집 금지.

### TODO_LIST.md 포맷

```
| 주소 (Address) | 상태 (Status) | 작업 요약 (Summary) |
| :--- | :--- | :--- |
| W://root/frontend/search | ACTIVE | Command Palette 검색 구현 |
| W://root/frontend/dashboard | PENDING | 프로젝트 대시보드 화면 |
| W://root/backend/git_service | [BLOCKED] | Git 연동 (ref: cli_service 미완료) |
```

### WP_SNAPSHOT.json 포맷

```json
{
  "W://root/frontend/search": {
    "modTime": "2026-04-08T12:00:00",
    "size": 1234,
    "status": "S_PRG"
  }
}
```

### 상태 매핑

| WayPoint STATUS | TODO 상태 | 설명 |
| :--- | :--- | :--- |
| `S_IDL` | `PENDING` | 착수 전 |
| `S_PRG` | `ACTIVE` | 진행 중 |
| `S_STB` | (목록에서 제거) | 완료 — history에서 조회 |
| `S_ERR` / `S_REV` | `ACTIVE` | 수정/재검토 필요 |

### BLOCKED 판정

WayPoint CONNECTIONS.REFERENCE에 나열된 주소 중 STATUS가 S_STB가 아닌 항목이 있으면 `[BLOCKED]` 표시.
- 표시만 하며 작업을 차단하지 않음 (느슨한 관리)
- `todo list` 출력 시 BLOCKED 항목은 하단에 정렬

## CLI 명령

### `loadstar todo sync`

전체 WayPoint를 스캔하여 TODO_LIST를 동기화한다.

```bash
loadstar todo sync              # 전체 WP 스캔
loadstar todo sync W://주소     # 특정 WP만 개별 동기화
```

**동작:**
1. MAP 순회 → 전체 WayPoint 주소 수집
2. 각 WP 파일의 modTime/size를 WP_SNAPSHOT.json과 비교
3. 변경된 WP만 STATUS 재확인:
   - 신규 WP (스냅샷에 없음) → S_IDL/S_PRG면 TODO_LIST에 추가
   - STATUS 변경 → TODO_LIST 갱신 (S_STB면 제거)
   - 삭제된 WP → TODO_LIST에서 제거
4. WP_SNAPSHOT.json 갱신

### `loadstar todo list`

현재 TODO 목록을 출력한다.

```bash
loadstar todo list              # PENDING/ACTIVE/BLOCKED 항목 출력
```

### `loadstar todo history`

WayPoint TECH_SPEC의 완료 항목 `[x]`를 수집하여 최신순 출력.

```bash
loadstar todo history                  # 전체 WP에서 완료 항목 수집
loadstar todo history M://root/cli     # 해당 Map 하위 WP만 필터
```

**출력 예시:**
```
ADDRESS                          DATE        ITEM
W://root/frontend/app_shell      2026-04-08  리사이즈 패널 적용
W://root/frontend/map_view       2026-04-07  React Flow 기반 흐름도
```

## 삭제된 명령어

| 명령 | 삭제일 | 사유 |
| :--- | :--- | :--- |
| `todo add` | 2026-04-08 | sync가 WP STATUS 기반 자동 추가 |
| `todo update` | 2026-04-08 | sync가 STATUS 기반 자동 갱신 |
| `todo done` | 2026-04-08 | sync가 S_STB 감지 시 자동 완료 |
| `todo delete` | 2026-04-08 | sync가 삭제된 WP 감지 시 자동 제거 |
