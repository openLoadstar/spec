# 📋 TODO_SPEC — TODO 시스템 규격 (v2.0)

> ⚠️ **TODO 데이터(`.clionly/TODO/`)는 `loadstar todo` 명령으로만 조작한다.** 직접 편집 시 포맷이 깨질 수 있다.

## 데이터 포맷

### 테이블 구조

```
| 주소 (Address) | 발생 시간 (Time) | 작업 요약 (Summary) | 상태 (Status) | 선행 조건 (Depends_On) |
| :--- | :--- | :--- | :--- | :--- |
| W://경로/id | YYYY-MM-DD HH:MM | 작업 내용 요약 | PENDING | - |
```

### 컬럼 규칙

| 컬럼 | 형식 | 설명 |
| :--- | :--- | :--- |
| Address | `W://경로/id` | 작업 대상 WayPoint 주소. 항상 `W://` |
| Time | `YYYY-MM-DD HH:MM` | `loadstar todo add` 실행 시 자동 기입 |
| Summary | 한 줄 텍스트 | 상세 내용은 해당 `W://` 주소 참조 |
| Status | 아래 상태 코드 참조 | `loadstar todo update`로 변경 |
| Depends_On | `요소://경로/id` 또는 `-` | 선행 완료 조건. 없으면 `-` |

### 상태 코드

| 상태 | 설명 | 전이 방법 |
| :--- | :--- | :--- |
| `PENDING` | 착수 전 대기 | `loadstar todo add` 시 자동 부여 |
| `ACTIVE` | 진행 중 | `loadstar todo update [ADDRESS] ACTIVE` |
| `BLOCKED` | 선행 조건 미완료로 착수 불가 | `Depends_On` 기반 자동 계산 — `list` 출력 시 자동 표시. 수동 설정 불가. |
| `COMPLETED` | 완료 — 리스트에서 즉시 삭제됨 | `loadstar todo done [ADDRESS]` |

> `COMPLETED`, `FAILED`, `BLOCKED`는 `loadstar todo update`로 설정 불가. `BLOCKED`는 `Depends_On`에 의해 자동 판별된다.

## CLI 명령 요약

```bash
loadstar todo list                                        # PENDING/ACTIVE 항목 출력 (BLOCKED 자동 표시)
loadstar todo add [ADDRESS] "[SUMMARY]"                  # 새 항목 추가 (PENDING 상태)
loadstar todo add ... --depends [ADDRESS]                # 선행 조건 지정
loadstar todo update [ADDRESS] [STATUS]                  # 상태 변경 (PENDING / ACTIVE)
loadstar todo done [ADDRESS]                             # 완료 처리 (리스트에서 삭제)
loadstar todo delete [ADDRESS]                           # 취소/삭제 (리스트에서 삭제)
```

## 완료 이력 추적

TODO_LIST는 **현재 해야 할 작업만** 보관한다. 완료된 작업의 이력은 다음으로 추적한다:
- **WayPoint TECH_SPEC**: `[x] YYYY-MM-DD 항목명` 형태로 완료 날짜와 함께 보존
- **Git 이력**: WayPoint 파일의 git log로 변경 이력 조회 가능

### 운영 지침 (Operational Rules)
1. **최소 정보 원칙**: 상세한 기술 규격이나 CoT는 본 파일에 적지 않는다. 필요시 각 요소의 `W://` 주소를 직접 참조한다.
2. **완료 시 처리**: 상태가 `COMPLETED`로 변경된 항목은 본 리스트에서 즉시 삭제한다.
3. **AI 우선 순위**: 에이전트는 세션 시작 시 `LOADSTAR_MASTER_GUIDE.md` 다음으로 이 파일을 읽어 본인이 즉시 수행해야 할 `PENDING` 작업을 식별한다.
4. **Depends_On 규칙**: `Depends_On` 컬럼에 명시된 Address가 완료되지 않은 항목은 `[BLOCKED]` 상태로 간주한다. AI는 BLOCKED 항목을 착수 대상에서 제외하고, 선행 조건 완료 후 재평가한다. 선행 조건이 없으면 `-`로 표기한다.
5. **직접 편집 금지**: 테이블 데이터는 반드시 `loadstar todo` 명령으로만 수정한다. 직접 편집 시 `|` 구분 파싱이 깨질 수 있다.
