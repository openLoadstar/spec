# 📊 LOADSTAR Effectiveness Metrics

본 문서는 LOADSTAR 방법론의 효과를 측정하기 위한 지표 체계를 정의합니다.
측정 목적: 이 구조를 계속 사용할지 여부, 어느 부분을 개선할지를 데이터 기반으로 판단한다.

> 현재 상태: **지표 정의 단계** — 실제 측정 및 분석은 충분한 사용 이력이 쌓인 후 진행한다.

---

## 1. 측정 대상 영역

### M1. AI 재진입 비용 (Re-entry Cost)

> 핵심 가설: "WayPoint CODE_MAP 경유가 grep 직접 탐색보다 빠르다"

| 항목 | 측정 방법 |
| :--- | :--- |
| LOADSTAR 경유 재진입 시간 | 세션 시작 → 수정 대상 파일/라인 특정까지 교환된 메시지 수 또는 도구 호출 횟수 |
| 직접 탐색 재진입 시간 | grep/Glob으로 동일 목표 달성 시 호출 횟수 |
| 비교 기준 | 동일 작업 유형에서 두 방식의 호출 횟수 차이 |

### M2. CODE_MAP 정확도 (Drift Rate)

> 핵심 가설: "CODE_MAP이 실제 코드 위치를 올바르게 가리킨다"

| 항목 | 측정 방법 |
| :--- | :--- |
| scope 드리프트 | CODE_MAP scope 경로가 실제 존재하는지 여부 |
| 드리프트 발생 빈도 | SYNCED_AT 이후 해당 scope 내 파일에 commit이 발생한 횟수 |
| 신뢰 불가 임계 | 디렉토리 이동/삭제 또는 함수명 불일치 시 → `S_REV` 전환 권고 |

### M3. 메타데이터 유지 비용 (Maintenance Overhead)

> 핵심 가설: "메타데이터 관리 비용이 절감 효과보다 작다"

| 항목 | 측정 방법 |
| :--- | :--- |
| 메타데이터 작업 비율 | 전체 작업 중 WayPoint 생성·갱신에 쓰인 시간 또는 메시지 비율 |
| 갱신 누락 빈도 | 소스 수정 후 관련 WayPoint 미갱신 건수 (Hook 리마인더 무시 횟수) |
| 사후 복구 비용 | 조정 세션에서 정합성 복구에 소요된 작업 횟수 |

### M4. 컨텍스트 절감율 (Token Efficiency)

> 핵심 가설: "LOADSTAR 경유 시 불필요한 파일 로드가 줄어든다"

| 항목 | 측정 방법 |
| :--- | :--- |
| 경유 시 로드 파일 수 | WayPoint → CODE_MAP → 코드 경로에서 실제 Read한 파일 수 |
| 직접 탐색 시 로드 파일 수 | 동일 작업을 grep 탐색으로 진행했을 때 Read한 파일 수 |
| 절감 비율 | `(직접 - 경유) / 직접 × 100%` |

### M5. 작업 완결성 (Task Completeness)

> 핵심 가설: "WayPoint 체크박스로 중단 없이 작업이 이어진다"

| 항목 | 측정 방법 |
| :--- | :--- |
| 재착수 성공률 | 세션 재시작 후 이전 작업을 컨텍스트 없이 정확히 이어받은 비율 |
| OPEN_QUESTIONS 해소율 | 작업 완료 시점에 `[Q RESOLVED]`로 전환된 항목 비율 |
| TECH_SPEC 완주율 | 완료된 WayPoint 중 모든 체크박스가 `[x]`인 비율 |
