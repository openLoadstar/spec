# 🔍 LOADSTAR 메타 동기화 명세 (v2.0)

본 문서는 코드 수정 시 `.loadstar/` 메타데이터 갱신 누락을 감지하고 보정하는 방법을 정의합니다.

---

## 1. 문제 정의

AI 에이전트가 코드를 수정할 때, 관련 WayPoint의 TODO 체크박스나 SYNCED_AT이 함께 갱신되어야 하지만 누락되는 경우가 빈번하다.

### 누락 발생 원인
- 코드 수정에 집중하여 메타데이터 후처리를 잊음
- 세션이 종료되면서 후처리 작업이 중단됨
- 작업자가 매번 갱신 여부를 확인하는 것이 번거로움

### 일관성 모델: 허용 범위 내 일관성 (Tolerable Consistency)

LOADSTAR는 **완전 일관성을 목표로 하지 않는다.** 이는 의도된 설계 결정이다.

- 매 코드 수정마다 모든 메타데이터를 갱신하는 비용(토큰, 시간)은 그 효과를 초과한다
- 사람 중심 개발에서도 코드와 문서의 완전한 동기화는 사실상 불가능하다
- AI 세션 단절로 인한 후처리 누락은 구조적으로 발생할 수밖에 없다

**목표 상태: "드리프트를 알고 있는 상태"**

## 2. 해결 방안: 환경별 메타 동기화 전략

### 2.1 Claude Code (CLI 환경) — Hook 기반 리마인더

**메커니즘**: Claude Code의 `PostToolUse` Hook을 활용하여, 소스코드 편집 직후 메타데이터 갱신 리마인더를 자동 출력한다.

**구성 파일**:
```
프로젝트/
├── .claude/
│   ├── settings.json          # Hook 이벤트 설정
│   └── hooks/
│       └── loadstar-drift-check.sh  # 드리프트 감지 스크립트
```

**settings.json 설정**:
```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "bash \"$CLAUDE_PROJECT_DIR/.claude/hooks/loadstar-drift-check.sh\""
          }
        ]
      }
    ]
  }
}
```

**동작 흐름**:
```
AI가 소스코드 Edit/Write
    ↓
PostToolUse Hook 실행
    ↓
스크립트가 파일 경로 확인
    ↓
.loadstar/ 내부 파일이면 무시
    ↓
소스코드 파일이면 리마인더 출력 → Claude 대화창에 표시
    ↓
AI가 관련 WayPoint 갱신 여부 판단
```

**스크립트 필터링 규칙**:
- `.loadstar/` 하위 파일: 무시 (메타데이터 자체 수정)
- `.claude/` 하위 파일: 무시 (설정 파일)
- `go.mod`, `*.json`, `*.yaml`, `*.md` 등: 무시 (메타 갱신 불필요)
- 그 외 소스코드 파일: 리마인더 출력

**장점**:
- 별도 프로세스 불필요 (기존 `loadstar_monitor.exe` 대체)
- Claude 세션 내에서 즉시 동작
- 리마인더는 AI의 판단을 돕는 것이지 강제하지 않음 → 느슨한 관리 철학과 일치

### 2.2 IDE 환경 (Cursor, VS Code 등) — 규칙 파일 기반

IDE 환경에서는 `.cursorrules` 또는 Custom Instructions에 다음 규칙을 포함:

```
소스코드 수정 완료 후:
1. 관련 WayPoint의 TODO 체크박스 갱신 여부 확인
2. 세션 종료 전 `loadstar validate` 실행
```

> IDE별 Hook/자동화 메커니즘이 있다면 Claude Code와 유사한 방식으로 적용 가능.

### 2.3 수동 검증 — loadstar validate

모든 환경에서 공통으로 사용 가능한 CLI 명령:

```
loadstar validate
```

모든 WayPoint와 Map의 참조 링크를 추적하여 깨진 참조를 검출한다. 세션 종료 전 또는 주기적으로 실행하여 구조적 정합성을 확인한다.

## 3. 조정 세션 (Reconciliation Session)

드리프트가 누적되면 별도의 조정 세션을 통해 일괄 정합성을 맞춘다. 코드 구현 세션과 분리하여 운영한다.

**트리거 조건 (하나 이상 충족 시)**:
- SYNCED_AT 30일 초과 WayPoint가 존재하면서 관련 코드 변경 이력이 있는 경우
- `loadstar validate`에서 깨진 참조가 발견된 경우
- 사용자가 수동 조정 세션 요청

**진행 순서**:
1. `loadstar show`로 전체 WayPoint 상태 파악
2. `loadstar validate`로 깨진 참조 확인
3. SYNCED_AT 오래된 WayPoint 우선 갱신
4. WayPoint TODO 체크박스 상태 현행화
5. 완료 후 SYNCED_AT 날짜 갱신
