# 📝 LOADSTAR Unified Element Format

---

## Map vs WayPoint 역할 구분

| 역할 | Map (M://) | WayPoint (W://) |
| :--- | :--- | :--- |
| 목적 | WayPoint 탐색을 위한 구조적 길잡이 | 모든 작업의 실행 단위 |
| WAYPOINTS | 하위 Map 또는 WayPoint 목록 | — |
| CONNECTIONS | — | PARENT, CHILDREN, REFERENCE |
| CODE_MAP | — | 코드 수정 시 탐색 범위 지정 (선택) |
| TODO | — | 실제 작업 명세 |

**원칙**: CLI 명령(`loadstar todo`, `loadstar create` 등)의 작업 단위는 항상 WayPoint 기준으로 추적한다. Map은 WayPoint에 도달하기 위한 계층 경로 역할만 한다.

---

## WayPoint 구조 규약

- WayPoint는 하위 WayPoint를 가질 수 있다. 하위 WayPoint는 `CONNECTIONS.CHILDREN`에 `W://` 주소로 등록한다.
- CODE_MAP은 선택 사항이다. 코드 수정이 수반되는 WayPoint에만 작성하며, 순수 계획·조율·문서 작업만 다루는 WayPoint는 생략할 수 있다.
- CODE_MAP이 없는 WayPoint에서 코드 작업이 새로 발생하면 그 시점에 CODE_MAP 섹션을 추가한다.

## WayPoint 포맷

```
<WAYPOINT>
## [ADDRESS] W://경로/ID
## [STATUS] SCHEMA_DEF_기준_코드

### IDENTITY
- SUMMARY: 요소의 목적 및 정체성 요약
- METADATA: [버전, 생성일, 우선순위 등]
- SYNCED_AT: YYYY-MM-DD HH:mm (마지막으로 실제 상태와 동기화한 시각)

### CONNECTIONS
- PARENT: 부모 요소 주소 (예: M://..., W://...)
- CHILDREN: 하위 WayPoint 주소 리스트 (예: W://...)
- REFERENCE: 참조 관계 주소 리스트 (예: W://...)

### CODE_MAP (선택 — 코드 수정이 수반되는 경우에만)
- scope: 탐색 범위 디렉토리 경로 (복수 가능)
  - 예: backend/src/main/java/com/loadstar/explorer/service/
  - 예: frontend/src/features/waypoint-editor/
- AI는 scope 범위 내에서만 grep/탐색을 수행하여 수정 대상을 특정한다.

### TODO
- ADDRESS: W://작업_대상_WayPoint_주소
- SUMMARY: 작업 내용 요약
- TECH_SPEC: 작업 수행에 필요한 기술적 요구사항 및 제약 조건
  - 항목은 두 가지 유형으로 구분한다:

  #### TASK 항목 (일회성 구현 작업)
    - [ ] 미완료 항목
    - [x] YYYY-MM-DD 완료된 항목 (완료 날짜 기입)
    - AI 재진입 시 체크박스 상태로 "어디서 멈췄는가"를 즉시 파악한다.
    - 완료 항목이 과다하게 누적되면 아카이브 파일로 분리할 수 있다. (아래 "TECH_SPEC 아카이브" 참조)

  #### RECURRING 항목 (반복 수행 작업)

  **목적**: TASK는 1회성 구현이며 완료되면 `[x]`로 고정된다. 반면 테스트·빌드·린트·포맷팅처럼 **코드가 바뀔 때마다 다시 수행해야 하는 작업**은 별도로 표기하여, AI가 세션마다 자동 재수행 대상을 구분하도록 한다.

  **표기**: `- (R) 작업 설명`

  **전형적 용례**:
    - `(R) 변경 후 mvn test 실행`
    - `(R) 변경 후 npm run build로 컴파일 검증`
    - `(R) ESLint 경고 0개 유지`
    - `(R) API 변경 시 OpenAPI 스펙 동기화`
    - `(R) 문자열 변경 시 i18n 키 정합성 검증`

  **TASK와의 차이**:

  | 속성 | TASK (`[ ]` / `[x]`) | RECURRING (`(R)`) |
  |:---|:---|:---|
  | 완료 개념 | 있음 (1회 체크로 고정) | 없음 (매번 재수행) |
  | S_STB 판정 | TASK 전부 `[x]`면 S_STB | 무관 |
  | CLI 표시 (`todo list` 등) | 표시 대상 | 표시 제외 |
  | AI 처리 | 착수 전 확인 + 완료 시 체크 | 코드 수정 시점마다 자동 실행 |
  | 제거 시점 | 일반적으로 유지 (이력) | 해당 검증이 더 이상 필요 없어진 경우 |

  **추가 기준**:
    - 해당 WP 산하 코드 변경 시 **반복적으로 수행해야 할 검증·의식**이 발견된 시점에 추가한다.
    - 일회성 검증은 TASK로 등록한다(예: `[ ] 초기 DB 마이그레이션 스크립트 실행`).

  **제거 기준**:
    - 해당 검증이 프로젝트 전역 자동화(pre-commit Hook, CI 파이프라인)로 이관되어 WP 수준 리마인더가 불필요해진 경우.
    - 한 WP에 `(R)` 항목이 5개 이상 누적되면, 그 WP의 스코프가 너무 넓거나 CI로 옮겨야 할 징후이므로 재검토한다.

  **AI 실행 프로토콜**:
    1. TECH_SPEC의 TASK 항목을 `[x]`로 체크하는 시점에,
    2. 같은 WP의 모든 `(R)` 항목을 순차적으로 실행한다.
    3. 실패 시 `loadstar log add [WP] ISSUE "..."`로 기록하고 작업을 멈춘다.
    4. 성공은 별도 체크하지 않는다(다음 수정 시 다시 실행될 것이므로).

  **주의**:
    - RECURRING은 **WP 단위** 검증이다. 프로젝트 전역 검증(예: "모든 PR에 테스트 필수")은 CI·Hook·pre-commit에서 강제한다.
    - WP 스코프가 좁을수록 `(R)` 항목이 적어지는 것이 정상이다.

### ISSUE
- 설계 시점 알려진 제약/미결 문제
- OPEN_QUESTIONS: 작업 착수 전 사람의 판단이 필요한 미결 결정 사항
  - 형식: `[Q1] 질문 내용`
  - 결정 완료 시: `[Q1 RESOLVED] 결정 내용`으로 인플레이스 업데이트
  - AI는 미해결 OPEN_QUESTIONS가 있으면 작업 착수 전 반드시 사람에게 확인을 요청한다.

### COMMENT
- 자유 형식 코멘트
</WAYPOINT>
```

## TECH_SPEC 아카이브 규칙

WayPoint의 완료 항목이 과다하게 누적되어 토큰 비용이 증가하는 경우, 완료 항목을 별도 파일로 분리할 수 있다.

- **아카이브 파일 위치**: `.loadstar/WAYPOINT/` 디렉토리 내, 원본과 같은 위치
- **명명 규칙**: `원본파일명-archive-NN.md` (NN은 순번)
  - 예: `root.frontend.waypoint_editor-archive-01.md`
- **분리 시점**: S_STB 전환 시 또는 완료 항목이 30개 이상 누적 시
- **아카이브 파일 내용**: 완료된 TECH_SPEC 항목만 포함 (날짜 포함)
- **WayPoint COMMENT에 참조 기록**: `아카이브: root.frontend.waypoint_editor-archive-01.md`
- **AI 프로토콜**: 아카이브 파일은 일상 작업 시 로드하지 않는다. 과거 이력 조회가 필요할 때만 참조한다.


## LOADSTAR_INIT.md 포맷

세션 진입점 파일. AI가 작업 시작 전 프로젝트 컨텍스트를 복원하기 위해 읽는 단일 문서이다.

- **위치**: `.loadstar/LOADSTAR_INIT.md`
- **용도**: AI 세션 시작/종료 시 참고사항을 남기는 프로젝트 레벨 메모
- **작성 주체**: AI 또는 사람이 자유롭게 편집 (UI 대시보드에서 편집 가능)
- **로드 시점**: 세션 시작 시 CLAUDE.md 다음으로 읽는다

```
# LOADSTAR_INIT — {프로젝트명}

> AI 세션 진입 시 이 파일을 먼저 읽어 프로젝트 컨텍스트를 복원합니다.

## 프로젝트 개요

- **스택**: (기술 스택 요약)
- **서비스 URL**: (주요 URL)

## AI 참고사항

> 세션 시작/종료 시 AI가 남기는 메모. 작업 방침, 주의사항, 다음 세션 인수사항 등.

- (자유 형식)

## WayPoint 상태 요약 (YYYY-MM-DD 기준)

(선택 — 수동 또는 자동 갱신)
```

**원칙**:
- 자유 형식 마크다운이며, 위 구조는 권장 템플릿이다.
- 고정 규칙은 CLAUDE.md에, 변동 상태는 LOADSTAR_INIT.md에 기록한다.
- AI는 이 두 파일만으로 작업 맥락을 완전히 복원할 수 있어야 한다.
