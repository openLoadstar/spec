# 📋 AVCS Standard Schema Definitions

## 1. Status Codes

> **적용 대상: WayPoint 전용.** Map은 STATUS를 갖지 않는다 (인덱스 역할만 수행). 진행 상태는 산하 WayPoint가 단일 출처(Single Source)이며, Map은 산하 WP 목록만 관리한다. (`05.ELEMENT_FORMAT.md` Map 포맷 참조)

- `S_IDL`: Idle (대기)
- `S_PRG`: In Progress (진행 중)
- `S_STB`: Stable (완료/안정)
- `S_ERR`: Error (오류 발생)
- `S_REV`: Review Required (검토 필요)

## 2. Link Types
- `L_REF`: Reference (참조)
- `L_SEQ`: Sequential (선후 관계)
- `L_TST`: Test Relation (검증 관계)

## 3. TODO Status Codes
- `PENDING`: 대기 중 (착수 전)
- `ACTIVE`: 진행 중
- `BLOCKED`: 선행 조건(`Depends_On`) 미완료로 착수 불가
- `COMPLETED`: 완료 (TODO_LIST에서 즉시 삭제)
- `FAILED`: 실패 (원인 기록 후 재등록 또는 취소 결정)

## 4. OPEN_QUESTIONS 상태
- `[Q{N}]`: 미결 — AI는 해당 항목이 있으면 작업 착수 전 사람에게 확인을 요청한다.
- `[Q{N} DEFERRED]`: 보류 — 결정을 당장 내릴 수 없거나 뒤로 미룬 상태. CLI `question` 출력에는 표시되지만 차단 요소로는 취급되지 않는다.
- `[Q{N} CONFIRMED]` 또는 `[Q{N} CONFIRMED <DECISION_ID>]`: 확인 완료 — 사람이 결정을 승인한 상태. 기본 출력에서 제외된다(`question --all`로 조회). 설계적 판단은 Decision 파일로 기록하고 ID를 표기하며, 단순 Q&A는 인라인으로만 처리해도 된다. 구분 기준은 `05.ELEMENT_FORMAT` Decision 섹션 참조.

**형식 예시**
```
### ISSUE
- OPEN_QUESTIONS:
  - [Q1] MAP 삭제 시 그 맵 산하 모든 WP를 어떻게 처리할 것인가?
  - [Q2 DEFERRED] 차트 라이브러리 최종 선택 — 성능 벤치 후 재검토
  - [Q3 CONFIRMED orphan_delete.2026-04-24.001] Block + --cascade 플래그
  - [Q4 CONFIRMED] 인라인으로 합의된 단순 결정
```

**규약**
- `<DECISION_ID>`는 `.loadstar/DECISIONS/` 하위 파일명에서 확장자(`.md`)를 뺀 형태.
- Decision 파일 포맷은 `05.ELEMENT_FORMAT.md` Decision 섹션 참조.
- 한 Decision이 여러 WP에 영향을 주면, 각 WP의 해당 Q 라인 모두 같은 DECISION_ID를 참조한다.

## 5. TECH_SPEC 체크박스 규칙
- `- [ ] 항목`: 미완료
- `- [x] YYYY-MM-DD 항목`: 완료 (완료 날짜 기입)
- AI 재진입 시 체크박스 상태로 중간 진행 위치를 즉시 파악한다.
- 체크박스 업데이트는 `loadstar edit` 또는 직접 편집으로 수행하며, 변경 이력이 Git에 자동 기록된다.
- 완료 항목이 과다하게 누적된 경우, WayPoint 아카이브 파일로 분리할 수 있다 (`05.ELEMENT_FORMAT.md` 참조).

## 6. SYNCED_AT 규칙
- WayPoint IDENTITY 및 Map IDENTITY에 포함되는 날짜 필드.
- **의미**: 이 요소의 내용이 실제 코드/상태와 마지막으로 일치함을 확인한 날짜.
- **갱신 시점**:
  - WayPoint: TECH_SPEC 항목 체크, CODE_MAP 갱신, 또는 STATUS 변경 시
  - Map: 하위 요소 구성이 변경될 때
- **AI 행동 규칙**: SYNCED_AT이 현재 날짜 기준 30일 이상 지난 WayPoint는 CODE_MAP scope의 유효성을 재확인한 후 작업한다.
- 형식: `YYYY-MM-DD HH:mm` (시/분 포함. 하위 호환: 기존 `YYYY-MM-DD` 형식도 유효)
